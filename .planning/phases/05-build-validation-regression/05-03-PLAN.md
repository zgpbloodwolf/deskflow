  <what-built>GUI 配置界面冒烟测试 (per D-08, D-09, BQ-03)</what-built>
  <how-to-verify>
    在 Mac 上执行以下 GUI 冒烟测试：

    **SC-3a: GUI 窗口打开**
    1. 双击打开 build/bin/Deskflow.app（或在终端运行 `open build/bin/Deskflow.app`）
    2. 验证：主窗口出现，标题栏显示 "Deskflow"
    3. 验证：状态栏显示当前连接状态

    **SC-3b: 设置界面显示**
    1. 点击菜单栏 File -> Settings（或快捷键 Cmd+,）
    2. 验证：Settings 对话框弹出
    3. 验证：设置界面显示所有配置选项（无 TLS/加密相关选项，per Phase 2 清理）
    4. 验证：界面显示正常，无布局错乱或控件缺失

    **SC-3c: Client 模式配置**
    1. 在 Settings 中选择 "Client" 模式
    2. 输入服务器 IP 地址（例如 Windows 机器的局域网 IP）
    3. 点击 Save/OK 保存设置
    4. 验证：设置已保存，重新打开 Settings 确认 IP 地址正确

    **SC-3d: Server 模式配置**
    1. 在 Settings 中选择 "Server" 模式
    2. 验证：显示屏幕布局配置界面
    3. 可以拖拽调整屏幕位置
    4. 点击 Save/OK 保存设置

    **SC-3e: 核心进程启动**
    1. 在主窗口点击 "Start" 按钮
    2. 验证：状态变为 "Running" 或 "Connected"
    3. 验证：deskflow-core 子进程已启动（`ps aux | grep deskflow-core`）
    4. 点击 "Stop" 按钮
    5. 验证：deskflow-core 子进程已终止

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>剪贴板共享和文件拖拽传输冒烟测试 (per D-08, D-10, BQ-04, BQ-05)</what-built>
  <how-to-verify>
    前提条件：Mac 和 Windows 通过局域网连接，一台作为 Server 一台作为 Client，Deskflow 正在运行且鼠标可以穿越屏幕边界。

    **SC-4a: 文本剪贴板 Mac -> Windows**
    1. 在 Mac 上复制一段文本（Cmd+C），内容为 "Deskflow clipboard test - Hello World!"
    2. 将鼠标移到 Windows 屏幕上
    3. 在 Windows 上粘贴（Ctrl+V）
    4. 验证：粘贴的文本与复制的内容完全一致

    **SC-4b: 文本剪贴板 Windows -> Mac**
    1. 在 Windows 上复制文本 "Windows to Mac clipboard test"
    2. 将鼠标移到 Mac 屏幕上
    3. 在 Mac 上粘贴（Cmd+V）
    4. 验证：粘贴的文本与复制的内容完全一致

    **SC-4c: 图片剪贴板 Mac -> Windows**
    1. 在 Mac 上截图（Cmd+Shift+4 选择区域）或从浏览器复制图片
    2. 将鼠标移到 Windows 屏幕上
    3. 在 Windows 上粘贴（Ctrl+V）到画图或文档中
    4. 验证：图片正确显示，无变形

    **SC-4d: 图片剪贴板 Windows -> Mac**
    1. 在 Windows 上使用截图工具或从浏览器复制图片
    2. 将鼠标移到 Mac 屏幕上
    3. 在 Mac 上粘贴（Cmd+V）到预览或其他应用
    4. 验证：图片正确显示

    **SC-5a: 文件拖拽 Mac -> Windows**
    1. 在 Mac 桌面上创建一个测试文件（例如 test-file.txt，内容 "drag test content"）
    2. 从 Mac 桌面拖拽文件，越过屏幕边界到 Windows 桌面
    3. 验证：文件出现在 Windows 桌面上
    4. 打开文件验证内容完整

    **SC-5b: 文件拖拽 Windows -> Mac**
    1. 在 Windows 桌面上创建一个测试文件
    2. 从 Windows 桌面拖拽文件，越过屏幕边界到 Mac 桌面
    3. 验证：文件出现在 Mac 桌面上
    4. 打开文件验证内容完整

    **SC-5c: 文件内容完整性**
    1. 计算原始文件的 MD5/SHA256
    2. 计算传输后文件的 MD5/SHA256
    3. 验证：两个哈希值完全一致

    如果任何测试失败 (per D-10)，描述具体现象（哪个方向失败、错误信息、日志输出），Claude 会定位并修复后重新测试。
  </how-to-verify>
