  替换 cursor-mark sentinel 为类型安全的哨兵（per D-05, D-10）：
  ```cpp
  // 替换前（行 101）:
  ISocketMultiplexerJob *m_cursorMark = nullptr;

  // 替换后: 使用静态哨兵地址，但通过 CursorMarkTag 消除 reinterpret_cast
  // 在 SocketJobs = std::list<ISocketMultiplexerJob*> 的设计中，dummy 节点
  // 需要一个唯一地址来标识。用 CursorMarkTag 静态实例提供该地址。
  struct CursorMarkTag {};
  static CursorMarkTag s_cursorMarkTag;
  ISocketMultiplexerJob *m_cursorMark = reinterpret_cast<ISocketMultiplexerJob*>(&s_cursorMarkTag);
  ```

  **注意：** CursorMarkTag 方案仍使用 reinterpret_cast 获取唯一地址，但该地址永远不会被解引用（与原代码行为一致），且语义更清晰。如果需要完全消除 reinterpret_cast，可以用一个专门的 `ISocketMultiplexerJob` 子类作为哨兵：
  ```cpp
  // 备选方案：零 reinterpret_cast
  class CursorMarkSentinel : public ISocketMultiplexerJob {
  public:
    ISocketMultiplexerJob *run(bool, bool, bool) override { return nullptr; }
    ArchSocket getSocket() const override { return nullptr; }
    bool isReadable() const override { return false; }
    bool isWritable() const override { return false; }
  };
  // 成员: std::unique_ptr<CursorMarkSentinel> m_cursorMarkSentinel;
  // 使用 m_cursorMarkSentinel.get() 作为哨兵值
  ```
  推荐使用 CursorMarkSentinel 方案彻底消除 reinterpret_cast。
