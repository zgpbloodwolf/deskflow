**Step C: Create src/lib/net/AsioTCPSocket.h**

New file -- header-only class skeleton implementing IDataSocket. Key design:
- Per D-04: each AsioTCPSocket owns its own `asio::io_context` and runs it in a dedicated `std::thread`
- Per D-06: completion handlers post events to EventQueue via `m_events->addEvent()`
- Per D-03: use `asio::steady_timer` for timeouts
- Use `asio::strand` to serialize async operations within the io_context
- Use `executor_work_guard` to keep io_context::run() alive

Class structure:
```cpp
#pragma once

#include "io/StreamBuffer.h"
#include "net/IDataSocket.h"
#include "net/NetworkAddress.h"

#include <asio.hpp>

#include <atomic>
#include <memory>
#include <mutex>
#include <thread>

class IEventQueue;

class AsioTCPSocket : public IDataSocket, public std::enable_shared_from_this<AsioTCPSocket>
{
public:
  explicit AsioTCPSocket(IEventQueue *events);
  AsioTCPSocket(IEventQueue *events, asio::ip::tcp::socket socket);
  ~AsioTCPSocket() override;

  // Disable copy/move
  AsioTCPSocket(const AsioTCPSocket &) = delete;
  AsioTCPSocket &operator=(const AsioTCPSocket &) = delete;

  // ISocket overrides
  void bind(const NetworkAddress &) override;
  void close() override;
  void *getEventTarget() const override;

  // IStream overrides
  uint32_t read(void *buffer, uint32_t n) override;
  void write(const void *buffer, uint32_t n) override;
  void flush() override;
  void shutdownInput() override;
  void shutdownOutput() override;
  bool isReady() const override;
  bool isFatal() const override;
  uint32_t getSize() const override;

  // IDataSocket overrides
  void connect(const NetworkAddress &) override;

private:
  void startAsyncRead();
  void onReadComplete(const std::error_code &ec, size_t bytesTransferred);
  void onWriteComplete(const std::error_code &ec, size_t bytesTransferred);
  void startAsyncWrite();
  void sendEvent(deskflow::EventTypes type);

  asio::io_context m_ioContext;
  asio::ip::tcp::socket m_socket;
  asio::strand<asio::io_context::executor_type> m_strand;
  asio::executor_work_guard<asio::io_context::executor_type> m_workGuard;
  std::thread m_ioThread;

  IEventQueue *m_events;
  mutable std::mutex m_mutex;

  StreamBuffer m_inputBuffer;
  StreamBuffer m_outputBuffer;

  std::array<char, 4096> m_readBuffer{};
  std::atomic<bool> m_connected{false};
  std::atomic<bool> m_writable{false};

  // flush() synchronization -- uses promise/future pattern
  std::mutex m_flushMutex;
  std::condition_variable m_flushCV;
  std::atomic<bool> m_flushed{true};
};
```

**AsioTCPSocket.cpp implementation details:**
- Constructor: initialize io_context, socket, strand, work_guard. Start io_context thread with `io_context::run()`.
- Destructor: call `close()`, then `m_ioContext.stop()`, then `m_ioThread.join()`.
- `connect()`: use `asio::ip::tcp::resolver::async_resolve` then `asio::async_connect` on strand. On success: set TCP_NODELAY, keep_alive, set m_connected=true, post DataSocketConnected event, start async read. On failure: post DataSocketConnectionFailed event.
- `startAsyncRead()`: `m_socket.async_read_some` on strand with 4096 byte buffer. On data: write to m_inputBuffer, post StreamInputReady event, chain next read. On EOF/error: post SocketDisconnected event.
- `write()`: copy data to m_outputBuffer under mutex, then `asio::post(m_strand, ...)` to trigger startAsyncWrite on io_context thread.
- `startAsyncWrite()`: under mutex, get output buffer data, call `asio::async_write` on strand. On complete: pop written data from m_outputBuffer, check if fully flushed, notify flush CV.
- `flush()`: wait on m_flushCV until m_flushed is true (with timeout to prevent deadlock).
- `read()`: copy from m_inputBuffer under mutex (same pattern as TCPSocket::read).
- `close()`: cancel all async ops by closing socket, post SocketDisconnected if was connected.
- `bind()`: open acceptor, bind to address (for listen socket usage if needed).
- `isReady()`: return `m_inputBuffer.getSize() > 0`.
- `getSize()`: return `m_inputBuffer.getSize()`.
- `isFatal()`: return false (TCP sockets, same as TCPSocket).
- `getEventTarget()`: return `const_cast<void*>(static_cast<const void*>(this))`.

Use `LOG_DEBUG`, `LOG_WARN` macros from `base/Log.h` for logging. Use `#include "base/IEventQueue.h"` for EventQueue access.

**AsioTCPListenSocket.cpp implementation details:**
- Owns `asio::io_context` + `asio::ip::tcp::acceptor`
- Runs io_context in dedicated thread with work_guard
- `bind()`: open acceptor, set SO_REUSEADDR on Unix, bind, listen
- `accept()`: synchronous wrapper -- on incoming connection, create AsioTCPSocket with accepted socket. Uses async_accept internally: when connection arrives, post ListenSocketConnecting event, store accepted socket. `accept()` returns the stored socket.
- `close()`: close acceptor, stop io_context
- `getEventTarget()`: return `const_cast<void*>(static_cast<const void*>(this))`

- Constructor: `AsioTCPSocketFactory(IEventQueue *events)` (no SocketMultiplexer needed)
- `create()`: return `new AsioTCPSocket(m_events)` -- ignores SecurityLevel parameter (TLS is Phase 2)
- `createListen()`: return `new AsioTCPListenSocket(m_events)` -- ignores SecurityLevel parameter

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Network -> AsioTCPSocket | Untrusted TCP data arrives via async_read_some. Data is written to StreamBuffer before parsing. |
| Asio io_context thread -> EventQueue | Asio completion handlers run on io_context thread and call EventQueue::addEvent(). EventQueue must be thread-safe for this. |
| EventQueue thread -> AsioTCPSocket::write() | Upper-layer code (Server/ClientProxy) calls write() from EventQueue thread. Data goes to m_outputBuffer, then asio::post to io_context thread. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | D (Denial of Service) | AsioTCPSocket::onReadComplete | accept | Input buffer capped at 1MB (same as TCPSocket's s_maxInputBufferSize), excess data silently dropped |
| T-01-02 | I (Information Disclosure) | AsioTCPSocket plaintext TCP | accept | Phase 2 removes TLS; plaintext is accepted per project scope (LAN only) |
| T-01-03 | T (Tampering) | Asio strand serialization | mitigate | All async operations bound to strand, preventing concurrent handler execution on same socket |
</threat_model>
